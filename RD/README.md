### 要件定義とは
- システムに求められる機能や要件を明確にすること
- クライアントとエンジニアの認識のずれをなくすこと
  - エンジニア同士の認識のずれをなくすこと
- 機能要件と非機能要件を定義する
  - 機能要件：システムが具体的に何をするか
  - 非機能要件：パフォーマンス、セキュリティ、ユーザビリティなど
- 要求の収集（クライアントからのヒアリング）、それを整理・分類して要件定義書を作成する
- 明確さ、一貫性、実現可能性を意識すること
